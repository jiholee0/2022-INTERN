- NCSA(아파치 서버)
	 NCSA(National Center for Supercomputing Applications)계열의 서버에서 사용하는 파일형식으로 CLF(Common Logfile Format)라고도 불린다. 웹 서버마다 자체적으로 로그파일 형식을 제안하지만, 대부분의 웹 서버 제작사는 표준로그파일형식(CLF)을 지원한다. 표준로그파일은 Transfer 또는 Access Logfile 이라 불리며, 파일이름은 access_log와 같이 정하고 있다.

172.21.13.45 - Microsoft\JohnDoe [07/Apr/2004:17:39:04 -0800] 
"GET /scripts/iisadmin/ism.dll?http/serv HTTP/1.0" 200 3401

필드					| 설명

원격 호스트 주소				| 요청한 클라이언트의 IP 주소입니다.
원격 로그 이름				| 사용되지 않습니다. 이 값은 항상 하이픈입니다.
사용자 이름				| 서버에 액세스한 인증된 사용자의 이름입니다. 익명 사용자는 하이픈으로 표시됩니다. 가장 좋은 방법은 애플리케이션이 항상 사용자 이름을 제공하는 것입니다.
날짜, 시간 및 그리니치 표준시(GMT) 오프셋	| 활동이 발생한 현지 날짜 및 시간입니다. 그리니치 표준시로부터의 오프셋도 표시됩니다.
요청 및 프로토콜 버전			| 클라이언트가 사용한 HTTP 프로토콜 버전입니다.
서비스 상태 코드				| HTTP 상태 코드입니다. (값 200은 요청이 성공적으로 완료되었음을 나타냅니다.)
보낸 바이트				| 서버에서 보낸 바이트 수입니다.

LogFormat "%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\"" combined


로그 데이터 수집 방법
	1. 웹 서버에서 제공하는 로그 수집
	2. 세션을 이용하여 로그 수집
	3. 스크립트 코드를 삽입하여 수집 - 모든 페이지마다 코드 삽입해야함. 원하는 데이터만 얻을 수 있다는 장점이 있지만, 다른 데이터를 얻고 싶어지면 수정하기 힘듦.

방문객이 웹 서버에 접속할 때 발생되는 세션 정보를 이용하여 방문자 세션을 구분하고, 세션 정보가 존재하지 않는 로그에 대해서는 최대 유휴 시간을 적용하여 구분한다.

whois 정보를 이용하여 방문 ip 주소에 대한 정보 얻을 수 있음.
